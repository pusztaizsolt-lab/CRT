03 Belépés (PIN + 2FA)

1
Felhasználónév + PIN megadása
6 jegyű numerikus PIN a cellákba. Kattints: Belépés
2
OTP kód emailben
6 jegyű egyszeri kód érkezik a regisztrált email-re (10 perc érvényes). Add meg, kattints: Ellenőrzés
3
Belépve → Főmenü
A JWT token 8 óráig érvényes. Kilépés után új PIN + OTP szükséges.
Brute force védelem: 4 hibás PIN kísérlet → 30 perces automatikus zárolás. Feloldás: Admin konzol → felhasználó → Feloldás.
Téma: A belépési oldalon 4 szín téma választható — az összes oldalon érvényes.

1

2
Script szerkesztés → lépések rögzítése (navigate / find / type / click / extract)

Teszt futtatás → egy termékre kipróbálja

Ütemezés: Beállítások → Webes árak → Scrape intervallum (alapért. 24 óra). Azonnali futtatás is elérhető.

15 Hibaelhárítás

A belépési oldal nem tölt be / "Szerver nem elérhető"
→ status.bat → fut-e a backend? → Ha nem: start.bat, várj 30 mp-et → http://localhost:8000/health
Az OTP kód nem érkezett meg
→ Spam mappa ellenőrzése → Beállítások → SMTP → Teszt küldés → Ha nincs SMTP: kód a szerver logban (logs\backend\backend.log)
Elfelejtett PIN
→ Admin konzol → felhasználó kikeresése → PIN reset
Fiók zárolva (brute force)
→ Várj 30 percet VAGY Admin konzol → Feloldás gomb
AI azonosítás nem működik / 503 hiba
→ Beállítások → Claude: Kapcsolat teszt → Ollama fut-e? → Legalább egy motornak működnie kell
LoRA tréning "befagyott" (pending státusz)
→ LoRA oldal → job sor → Törlés gomb → Indíts új tréninget
Export (xlsx/docx/PDF) nem tölt le
→ Böngésző letöltési blokkoló? → exports\ mappa létezik? → szerver log: logs\backend\backend.log
Scraper nem talál árakat
→ Webes árak → Ping teszt → elérhető a weboldal? → Script teszt futtatása → Playwright telepítve? (python -m playwright install chromium)

Diagnosztika oldal

Az összes szolgáltatás állapota egy helyen: ui/kezelőpult_v2.html — API, DB, Ollama, ChromaDB, platform info.

Első telepítés – helyes sorrend

1. setup.bat – Python 3.11 venv létrehozás, pip install, Playwright chromium

2. db_init.bat – PostgreSQL adatmappa inicializálás (automatikus port 5433 konfig)

3. migrate.bat – db_schema.py → admin user → v02–v08 migrációk

4. INDITAS.bat – backend indítás, böngésző: http://localhost:8000

Port konfiguráció

Miért 5433 és nem 5432?
Ha a gépen már fut telepített PostgreSQL (pl. pgAdmin-hoz), az foglalja az 5432-t. A CRT saját példánya ezért 5433-on fut — ez beágyazott, nem kell különválasztani.
Több gép / hálózati hozzáférés
→ .env: listen_addresses = '*' → PostgreSQL tűzfalszabály → ALLOWED_HOSTS bővítés a main.py-ban → HTTPS kötelező élesben!

iOS / mobil hozzáférés

A CRT backend REST API, így iPhone/iPad Safari-ból is elérhető — feltételek:

FeltételLeírásMegvalósítás
HTTPS iOS Safari csak HTTPS-en fut megbízhatóan (HTTP-n cookie/session problémák) Nginx reverse proxy + Let's Encrypt cert, vagy Cloudflare Tunnel
CORS A backend CORS-fejléceket kell visszaadjon az iOS domain felé main.py: allow_origins=["https://sajat.domain.hu"]
Hálózat iPhone ugyanazon a LAN-on legyen, VAGY nyilvános IP/domain Router port-forward: 443 → CRT gép:443 (Nginx)
PWA (opcionális) manifest.json + service worker → telepíthető iPhone-ra Tervezett v1.2 fejlesztés

Frissítés meglévő telepítésre

Hogyan frissítek új verzióra?
1. Állítsd le a backendet (Ctrl+C) · 2. Másold az új fájlokat (db, logs, .env NEM!) · 3. migrate.bat (migrációk idempotensek) · 4. Indítsd újra

Éles üzem előtt kötelező

⚠ Biztonsági ellenőrzőlista

K-1: DB jelszó – a crt2026 csak fejlesztésre! .env-ben cseréld le erős jelszóra és frissítsd a PostgreSQL felhasználót.

K-2: JWT secret – a crt_dev_secret_CHANGE_IN_PRODUCTION nem biztonságos! Legalább 32 véletlen karakter az .env-ben.

K-3: .env SOHA nem kerülhet szinkronba (Google Drive, Git) – tartalmaz minden titkot!
